Épica 2: Desarrollo (MVP)

Planificación de las funcionalidades clave que el Producto Mínimo Viable debe ofrecer para ser valioso y resolver los problemas identificados.

Para el Consultor

Funcionalidades diseñadas para agilizar el trabajo diario, mejorar la calidad de los entregables y facilitar el acceso al conocimiento colectivo.

Pregunta Clave de Diseño (General)

¿Cuál será la interfaz principal de la herramienta? ¿Será un chatbot integrado en Microsoft Teams, una aplicación web independiente o una combinación de ambas? Es crucial definir la experiencia de usuario desde el principio.

Historias de Usuario a Desarrollar:

2.1: Asistente para Especificaciones Técnicas

Un chatbot que asista en la traducción de requerimientos de cliente a un lenguaje técnico estandarizado (ABAP-BASIS).

Pregunta a resolver: ¿Qué estructura y secciones debe tener una "especificación técnica perfecta"? Necesitamos definir una plantilla estándar que la IA deberá seguir.

2.2: Búsqueda de Soluciones Pasadas

Un buscador que, usando palabras clave, proponga soluciones basadas en casos ya resueltos e identifique a las personas que trabajaron en ellos.

Pregunta a resolver: ¿Cómo se calificará la relevancia de los resultados? ¿Implementaremos un sistema de feedback (ej. "esta solución fue útil") para que el sistema aprenda y mejore?

2.4: Consulta Automatizada en SAP

Una interfaz con IA que permita buscar tablas y procedimientos en la base de datos de SAP de forma automática.

Pregunta a resolver: ¿Cómo se manejarán los permisos de seguridad? Es crítico asegurar que la IA solo pueda consultar datos autorizados y no afecte el rendimiento del sistema SAP productivo.

Para el Colaborador

Funcionalidades enfocadas en facilitar la contribución y el mantenimiento de la base de conocimiento para que se mantenga actualizada y relevante.

2.6: Contribución a la Base de Conocimiento

Un proceso sencillo para cargar nueva información y documentación a la base de conocimiento.

Pregunta a resolver: ¿Existirá un proceso de validación o curación antes de que un nuevo documento sea público para todos? ¿Quién será el responsable de revisar y aprobar las contribuciones?
