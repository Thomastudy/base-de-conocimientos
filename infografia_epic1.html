Épica 1: Planificación e Investigación

Definiendo los cimientos del proyecto para asegurar el éxito y la alineación con las necesidades del equipo.

1.1: Definición de Requisitos

Analizar requisitos para que la solución comprenda el entorno SAP y el lenguaje técnico ABAP, asegurando que las funcionalidades a automatizar sean las correctas.

Hitos Clave:

  • Realizar entrevistas con 3-5 consultores de distintos niveles.
  • Documentar los 5 casos de uso más frecuentes a resolver.
  • Crear un glosario de términos técnicos (ABAP-BASIS) para el sistema.

Pregunta Clave a Investigar

¿Cuáles son los criterios de aceptación para una "especificación técnica" generada automáticamente? ¿Qué nivel de detalle, formato y precisión se considera un éxito para que sea realmente útil?

1.2: Selección de Tecnología

Definir la arquitectura técnica final, priorizando la integración con Microsoft (Teams, SharePoint) y OpenAI. La arquitectura se basará en un modelo RAG utilizando SAP HANA Cloud (Vector Engine) como base de datos vectorial.

Hitos Clave:

  • Validar la viabilidad técnica del flujo SharePoint > SAP BTP > SAP HANA Cloud.
  • Realizar una Prueba de Concepto (PoC) del proceso completo de ingesta y consulta.
  • Definir el modelo de embedding a utilizar para la vectorización de documentos.

Preguntas Claves a Investigar

1. ¿Cuáles son los costos estimados de las APIs (OpenAI, SAP BTP) y cómo se monitoreará el consumo?
2. ¿Cómo se gestionará la seguridad y los permisos para acceder a documentos con información sensible?

1.3: Centralización del Conocimiento

Identificar y segmentar las áreas de conocimiento clave de la empresa para definir qué información alimentará el sistema, considerando tanto los servicios de consultoría como el software propio.

Áreas de Conocimiento ("Silos") Identificadas:

  1. Desarrollo Técnico y Plataforma SAP: Conocimiento sobre ABAP-BASIS, tablas, transacciones y desarrollos.
  2. Conocimiento Funcional (Servicios SAP): Experiencia de negocio en los distintos módulos de SAP.
  3. Productos de Software Propios: Documentación, guías y conocimiento técnico de las soluciones de software de la empresa.
  4. Gestión de Proyectos y Soluciones Implementadas: Repositorio de proyectos (servicios y software) para reutilización.
  5. Base de Casos de Soporte: Solución a errores comunes e issues recurrentes (servicios y software).
  6. Especificaciones Técnicas y de Requerimientos: Proceso de traducción de necesidades del cliente a lenguaje técnico.

Pregunta Clave a Investigar

¿Existen otros silos de conocimiento funcional relevantes además de los ya contemplados? Es necesario realizar un relevamiento para incluir áreas como Analytics o SAP Business One si corresponde.

Principales Cuellos de Botella a Resolver:

  • Resolución de Problemas Repetitivos: Evitar "reinventar la rueda" con issues y desarrollos ya solucionados.
  • Creación de Especificaciones Técnicas: Estandarizar y agilizar el proceso para mejorar la comunicación entre equipos.

1.4: Involucramiento de Expertos

Colaborar activamente con consultores experimentados para guiar el desarrollo con su conocimiento y asegurar que la herramienta se adapte a las necesidades reales del equipo.

Hitos Clave:

  • Agendar sesiones de validación semanales con los expertos designados.
  • Crear un canal de feedback directo (ej. en Teams) para consultas rápidas.
  • Grabar sesiones de resolución de problemas para transcribir y añadir al conocimiento.

Pregunta Clave a Investigar

¿Quiénes son los 2-3 consultores de referencia o "gurús" para cada uno de los silos de conocimiento definidos? Es crucial identificarlos para asegurar que el conocimiento extraído sea preciso, relevante y validado por expertos.
